## Lore Access Levels

### Level 1: Full Access
- All published lore
- All character profiles
- All world mechanics
- Sealed/unrevealed content
- **Products**: Main Game, Light Novels

### Level 2: Standard Access
- All published lore
- All character profiles
- Core world mechanics
- NO sealed content
- **Products**: Companion App, Music

### Level 3: Visual/Thematic Access
- Visual identity
- Character designs/names
- Basic world context
- NO detailed lore
- **Products**: Merchandise, Partnerships

### Level 4: Licensed Subset
- Pre-approved elements only
- Specific characters/concepts licensed
- No independent lore use
- **Products**: Partner Collaborations

## Content Type Guidelines by Product

### UI/UX Copy
```
Where: Main Game, Companion App
Voice: Functional but flavored
Length: Ultra-short (1-5 words for buttons, 10-20 for notifications)
Examples:
- Button: "Enter the Chase" not "Start Game"
- Notification: "Your idol needs Devotion" not "Daily reward available"
```

### Product Descriptions
```
Where: Merchandise Store
Voice: Evocative, desire-creating
Length: 50-150 words
Examples:
- "The Eternal Stage demands excellence. So does your wardrobe."
- "Ika's signature pink gradient, now yours to wear."
```

### Marketing Copy
```
Where: All products (social, ads, landing pages)
Voice: Full brand voice, DS-3 for CT
Length: Variable
Examples:
- See skills/ct-engagement.md for CT
- See skills/copywriting.md for general
```

### Narrative Content
```
Where: Light Novels, In-game story
Voice: Appropriate to POV character
Length: Variable
Examples:
- See character voice profiles
- See skills/character-voices.md
```

## Product Onboarding Protocol

When adding a new product:

1. **Classification**
   - Assign tier (1, 2, or 3)
   - Determine lore access level
   - Set voice adaptation range

2. **Documentation**
   - Add to this registry
   - Create product-specific guidelines if needed
   - Define content types and constraints

3. **Agent Briefing**
   - Update relevant agent files
   - Create product-specific quick reference
   - Establish review requirements

4. **Approval**
   - Human sign-off required for new products
   - Update CLAUDE.md if core workflow changes

## Cross-Product Consistency Rules

### Always Consistent
- 10 Inviolable Facts
- Character names and basic traits
- Visual identity (colors, aesthetic)
- Core brand pillars

### May Adapt
- Voice intensity (DS level)
- Content depth
- Terminology complexity
- Engagement style

### Never Cross-Contaminate
- Product-specific features in wrong context
- Unreleased information across products
- Conflicting lore statements
- Tone-inappropriate content
