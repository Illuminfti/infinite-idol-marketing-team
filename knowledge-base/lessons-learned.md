## Character Voice Lessons

### 2026-01-09 - Tilde (~) as Ika's Signature
**Category**: Character Voice
**Agent(s)**: Agent 02 (Content Strategist), Agent 09 (Resident Degen)
**Context**: Creating first Ika introduction tweet

**Discovery**: The tilde (~) at end of sentences is perfect signature move for Ika. It adds playfulness without cuteness, maintains shameless energy, and creates recognizable voice pattern.

**Quote**: "The tilde at the end is chef's kiss. Shameless but not desperate." - Agent 09

**Impact**:
- Future Ika content should use tilde strategically (not every sentence, but signature moments)
- Overuse would dilute power - use for emphasis/flirtation/playfulness
- Other characters should NOT use tilde (this is Ika's signature)

**Applied To**:
- Updated `ika-minami.md` character profile with tilde usage guideline
- Added to character-voices.md skill as "signature punctuation" technique

### 2026-01-09 - "Don't You Dare Look Away" Command Energy
**Category**: Character Voice
**Agent(s)**: Agent 02, Agent 09
**Context**: Ika introduction tweet

**Discovery**: Ika's voice works best with *command* energy, not *plea* energy. "Don't you dare look away" hits harder than "Please watch me" or "I hope you'll stay".

**Impact**:
- Future Ika CTAs should be commands/challenges, not requests
- Good: "Catch me if you can", "Try to keep up", "Watch this"
- Bad: "Please support me", "I hope you like it", "Maybe you'll watch?"

**Applied To**:
- Added "command vs plea" guideline to ika-minami.md
- Noted in content-creation.md skill

### 2026-01-09 - Existential Stakes Without Melodrama
**Category**: Character Voice
**Agent(s)**: Agent 02
**Context**: Ika personality tweets

**Discovery**: Ika can reference existential stakes ("keeps me existing") without being melodramatic. The key is matter-of-fact delivery - it's just how the world works for her.

**Impact**:
- Existential references should be casual, not overplayed
- Good: "These 47 fans keep me existing" (matter-of-fact)
- Bad: "Without you I'll DIE" (melodramatic)
- Balance: One existential reference per 3-5 tweets (not every tweet)

**Applied To**:
- Noted in ika-minami.md as "existential tone" guideline

## Content Format Lessons

### 2026-01-09 - 7-Tweet Threads Optimal for Lore Drops
**Category**: Content Format
**Agent(s)**: Agent 02, Agent 08
**Context**: Devotion system explainer thread

**Discovery**: 7-tweet threads hit sweet spot for lore explanations:
- Tweet 1: Hook (contrast with other games)
- Tweets 2-5: Explanation (mechanics, implications)
- Tweet 6: Character connection (make it personal)
- Tweet 7: Philosophical close (memorable ending)

**Why 7 Works**:
- Long enough for depth (not superficial)
- Short enough for completion (not exhausting)
- Odd number feels organic (not artificially structured)

**Impact**:
- Future lore drops should target 5-9 tweets (7 optimal)
- Under 5: Feels incomplete
- Over 10: Thread completion rate drops

**Applied To**:
- Added to content-creation.md as lore drop template

---

### 2026-01-09 - Batch Review for Similar Content is 85% Faster
**Category**: Process Efficiency
**Agent(s)**: Agent 08, Agent 09, Agent 00
**Context**: Ika personality tweet batch (5 tweets)

**Discovery**: Reviewing 5 similar tweets together (batch review) took ~3 hours total vs. ~10 hours if reviewed individually (5 x 2 hours each).

**Why Batching Works**:
- Common canon elements checked once
- Voice consistency verified across set (catch drift)
- Cultural tone assessed holistically
- Coordinator can see full week's content at once

**Impact**:
- Similar content should be created and reviewed in batches
- Good candidates: Character personality tweets, daily event posts, cosmetic reveals
- Bad candidates: Novel content, different content types mixed together

**Applied To**:
- Added batch review process to tiered-review-framework.md

## Technical / Platform Lessons

### 2026-01-09 - Character Count Strategy for Twitter
**Category**: Technical / Platform
**Agent(s)**: Agent 02
**Context**: Tweet creation

**Discovery**: Optimal tweet length is 180-240 characters (not 280 max). This leaves room for retweets with comments while maintaining readability.

**Why This Works**:
- Tweet 1 (Ika intro): 109 characters - High visibility, easy to RT with comment
- Longer tweets get less engagement (feel like "too much reading")
- But too short (<100) feels incomplete or low-effort

**Impact**:
- Target 180-240 characters for standalone tweets
- Under 150: Use for quick responses, not standalone content
- Over 250: Consider splitting into thread or cutting

**Applied To**:
- Added to content-creation.md platform guidelines

## Review Process Lessons

### 2026-01-09 - First Content Requires Full Review, Then Template Emerges
**Category**: Process
**Agent(s)**: Agent 00, Agent 08, Agent 09
**Context**: Ika introduction tweet vs personality batch

**Discovery**: First piece of new content type needs full comprehensive review (Tier 1). Once approved, it becomes template for Tier 2 streamlined reviews.

**Example**:
- TWEET-001 (Ika intro): Full review → 360 lines total
- Ika personality batch: Streamlined → ~150 lines total (used TWEET-001 as template)

**Impact**:
- Don't rush first examples - they set the standard
- Once template established, subsequent similar content can be Tier 2
- Capture first examples as "reference implementations"

**Applied To**:
- Added to tiered-review-framework.md
- Noted in canon-validation.md

## Agent Coordination Lessons

### 2026-01-09 - Agent 08 + Agent 09 Reviews Should Run in Parallel
**Category**: Process
**Agent(s)**: Agent 00
**Context**: Sequential vs parallel review analysis

**Discovery**: Running canon review (Agent 08) and cultural review (Agent 09) sequentially adds unnecessary wait time. They validate different things and can work simultaneously.

**Time Savings**:
- Sequential: 2 hours (canon) + 2 hours (cultural) = 4 hours
- Parallel: MAX(2 hours canon, 2 hours cultural) = 2 hours
- **50% time reduction**

**Impact**:
- All Tier 1 and Tier 2 content should have parallel reviews
- Task queue should create simultaneous review tasks
- Only Coordinator approval (Agent 00) needs to wait for both

**Applied To**:
- Documented in tiered-review-framework.md
- Will be implemented in automation system

## Red Flags to Watch For

These patterns indicate systemic issues:

**Approval Rate Drops Below 80%**:
- Agents aren't internalizing guidelines
- Guidelines are unclear or contradictory
- Review standards have shifted without documentation

**Same Mistake Repeated**:
- Learning not captured in this document
- Learning captured but not applied to guidelines
- Guidelines updated but agents not reading them

**Review Time Increasing**:
- Content complexity increasing (expected)
- Reviewers losing confidence (investigate)
- Process inefficiencies (optimize)

**Agent Conflicts Increasing**:
- Canon vs cultural authority unclear
- Tier classification disagreements
- Process ambiguity
